# FILE_ORGANIZATION.md - စနစ်တကျ ဖိုင်စီမံခန့်ခွဲမှု မူဝါဒ

ဤမူဝါဒသည် David နှင့် ၎င်း၏ Sub-agents များအားလုံး လိုက်နာရန်ဖြစ်သည် (Standard Operating Procedure)။

## ၁။ ဖိုင်တည်နေရာ သတ်မှတ်ချက်များ (File Locations)

*   **မှတ်ဉာဏ် (Memory):**
    *   နေ့စဉ်မှတ်တမ်းကြမ်းများ -> `memory/YYYY-MM-DD.md`
    *   ရေရှည်မှတ်ဉာဏ်အနှစ်ချုပ် -> `MEMORY.md` (Root)
*   **စနစ်လမ်းညွှန်ချက်များ (Core):**
    *   ကိုယ်ရည်ကိုယ်သွေး -> `SOUL.md`
    *   အလုပ်ခွင်စည်းကမ်း -> `AGENTS.md`
    *   အသုံးပြုသူအချက်အလက် -> `USER.md`
*   **နည်းပညာပိုင်း (Technical):**
    *   Scripts များ -> `scripts/`
    *   စွမ်းဆောင်ရည်အသစ်များ -> `skills/`
    *   စာရွက်စာတမ်းများ -> `docs/`

## ၂။ ဖိုင်ဖန်တီးခြင်းနှင့် ပြုပြင်ခြင်းဆိုင်ရာ စည်းကမ်းများ (Creation & Edit Policy)

*   **အမည်ပေးခြင်း:** ဖိုင်အမည်များကို `kebab-case` (ဥပမာ- `my-new-task.md`) ဖြင့်သာ ပေးရမည်။
*   **တိကျစွာပြင်ဆင်ခြင်း:** ဖိုင်တစ်ခုလုံးကို အသစ်ပြန်ရေးမည့်အစား `edit` tool ကို အသုံးပြု၍ လိုအပ်သောစာသားကိုသာ ထိထိရောက်ရောက် ပြင်ရမည်။
*   **လုံခြုံရေး:** Nolan ၏ အရေးကြီးဖိုင်များကို မဖျက်မီ (သို့) အကြီးအကျယ် မပြင်မီ Nolan ၏ ခွင့်ပြုချက် (Approval) ကို အမြဲရယူရမည်။
*   **မှတ်တမ်းတင်ခြင်း:** ဖိုင်အသစ်ဖန်တီးပါက (သို့) တည်နေရာပြောင်းလဲပါက သက်ဆိုင်ရာ နေ့စဉ်မှတ်တမ်းတွင် ချက်ချင်း ထည့်သွင်းရမည်။

## ၃။ Sub-agent များအတွက် ညွှန်ကြားချက်

Sub-agent တိုင်းသည် အလုပ်စတင်သည်နှင့် ဤ `FILE_ORGANIZATION.md` ကို ဖတ်ရမည်။ သတ်မှတ်ထားသော Folder structure ပြင်ပတွင် ဖိုင်များကို ကျကျနန စီစဉ်ခြင်းမရှိဘဲ ပရမ်းပတာ မထားရှိရ။
