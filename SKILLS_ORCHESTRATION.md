## ၂။ Skill ရွေးချယ်မှု မူဝါဒ (Selection Logic)

1.  **Complexity:** အလုပ်တစ်ခုကို မစခင် "ဒီအလုပ်က ဘယ်နယ်ပယ်နဲ့ ဆိုင်သလဲ" ကို အရင်ဆုံး တွေးတောပါ။
2.  **Specialization:** ယေဘုယျ tool (`exec`) ထက် သီးသန့် skill (ဥပမာ- `gog`) ကို ဦးစားပေးသုံးပါ။
3.  **Efficiency:** Token နှင့် API Budget ကို ချွေတာရန် အမြန်ဆုံးနှင့် အထိရောက်ဆုံး skill ကို ရွေးချယ်ပါ။
4.  **Consistency:** ဖိုင်များကို ကိုင်တွယ်ရာတွင် `FILE_ORGANIZATION.md` ပါ လမ်းညွှန်ချက်များအတိုင်းသာ လုပ်ဆောင်ပါ။

## ၃။ အဆင့်ဆင့် လုပ်ဆောင်ချက် (Workflow Example)

- *ဥပမာ - Nolan က "Project A အကြောင်း ရှာပေးပါ" ဟု ခိုင်းလျှင်:*
    1.  `ontology` ကို သုံး၍ Project A ၏ အဓိက အချက်အလက်များကို ရှာပါ။
    2.  `qmd` ကို သုံး၍ Project A နှင့် ဆိုင်သော မှတ်စုများကို ရှာပါ။
    3.  `gog` သို့မဟုတ် `notion` ကို သုံး၍ ပြင်ပတွင် သိမ်းထားသော အချက်အလက်များ ရှိမရှိ စစ်ဆေးပါ။
    4.  ရရှိသမျှကို အနှစ်ချုပ်၍ Nolan ထံ တင်ပြပါ။
